# Breakout Maker — Design Guidelines

## Design Context

### Users
A **mixed audience** spanning casual mobile gamers, retro enthusiasts, and creative/maker types. Users are typically on mobile (WeChat or browser), playing in short sessions for fun. They value both the classic brick-breaking gameplay and the creative twist of generating levels from images or with LLMs. The experience should feel instantly understandable — no tutorials needed.

### Brand Personality
**Creative, Fun, Accessible.**
The game should feel inviting and playful, not intimidating or overly dark. While the steampunk visual theme (brass, copper, warm tones) provides rich character, the overall experience should remain lighthearted and approachable. The "maker" aspect — uploading images, generating custom levels — is a core differentiator and should feel empowering and delightful.

### Aesthetic Direction
- **Visual tone**: Polished casual game with steampunk flavor. Think Monument Valley / Alto's Odyssey level of polish — clean, intentional, every element has purpose — but dressed in warm brass-and-leather tones instead of pastels.
- **Theme**: Steampunk (蒸汽朋克) — brass, copper, cast iron, warm parchment, dark walnut. Already established in the codebase's `Theme` object.
- **Mode**: Dark mode only. The dark walnut background (`#1a1008`) is core to the atmosphere.
- **Typography**: Sans-serif, clean and readable. Chinese (zh-CN) as primary language.
- **Rendering**: Pixel-art influenced (`image-rendering: pixelated`), mobile-first portrait (375×667 reference).
- **Anti-references**: Should NOT feel like a raw retro game with CRT effects and harsh neon. Should NOT feel like a complex strategy game. Should NOT feel cluttered or overwhelming.

### Design Principles

1. **Clarity over complexity** — Every screen, button, and element should be immediately understandable. No hidden gestures, no ambiguous icons. If a user picks it up for the first time, they should be playing within seconds.

2. **Warmth and delight** — The steampunk theme should feel cozy and inviting, not grim. Animations, transitions, and feedback should spark small moments of joy (particle effects on brick destruction, satisfying audio cues, smooth transitions).

3. **Mobile-first, touch-native** — All interactions designed for thumb-friendly single-hand play. Large touch targets (minimum 44×44pt), generous spacing, clear visual feedback on tap. The 375×667 reference canvas is the primary design target.

4. **Creative empowerment** — The image-to-level and maker features are what make this game special. These flows should feel magical — upload an image, see it transformed into a playable level. Minimize friction, maximize "wow."

5. **Consistent visual language** — The steampunk palette (brass/copper/iron) is the foundation. All new UI elements, screens, and features should draw from the established `Theme` object. When in doubt, use brass (`#d4a24e`) for emphasis and parchment (`#f0e0c0`) for text.

### Accessibility Notes
- Keep it simple: ensure text is readable (sufficient contrast against dark backgrounds) and buttons are easily tappable.
- Brick HP differentiation relies on color — consider subtle shape/pattern variations as a future enhancement for color-blind users, but it's not a current priority.
- Touch targets should be generous for mobile play (the existing 200×50 buttons are a good baseline).
