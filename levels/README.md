# 关卡配置文件格式

## 文件位置

所有关卡文件放在 `levels/` 目录下，文件名格式为 `level-NN.json`（如 `level-01.json`、`level-13.json`）。

构建脚本 `build.js` 会按文件名排序自动加载所有 `level-*.json` 文件。

## 构建

```bash
node build.js
```

运行后会自动：
1. 读取 `levels/` 下所有关卡 JSON 文件
2. 验证每个文件的格式
3. 注入到 `preview.html`

## JSON 格式

```json
{
  "name": "关卡13 - 你的关卡名",
  "gridWidth": 20,
  "gridHeight": 12,
  "ballSpeed": 320,
  "paddleWidth": 88,
  "lives": 3,
  "bricks": [
    { "row": 0, "col": 0, "hp": 1 },
    { "row": 0, "col": 1, "hp": 2 },
    { "row": 1, "col": 5, "hp": 999 }
  ]
}
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 关卡显示名称，格式建议 `"关卡N - 名称"` |
| `gridWidth` | number | 砖块网格列数（建议 16-30） |
| `gridHeight` | number | 砖块网格行数（建议 8-18） |
| `ballSpeed` | number | 球的速度（像素/秒），建议 280-400 |
| `paddleWidth` | number | 挡板宽度（像素），建议 70-120 |
| `lives` | number | 初始生命数，建议 3-5 |
| `bricks` | array | 砖块数组，每个砖块是一个对象 |

### 砖块字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `row` | number | 行号（从 0 开始，必须 < gridHeight） |
| `col` | number | 列号（从 0 开始，必须 < gridWidth） |
| `hp` | number | 生命值，决定需要几次打击才能摧毁 |

### HP 值对照表

| HP | 效果 | 视觉风格 |
|----|------|----------|
| `1` | 一击即碎 | 古铜色（Weathered bronze） |
| `2` | 需要 2 次打击 | 红铜色（Copper） |
| `3` | 需要 3 次打击 | 黄铜色（Brass） |
| `4` | 需要 4 次打击 | 红木色（Dark mahogany） |
| `999` | 不可摧毁 | 铸铁色（Cast iron），带角落螺栓 |

> **注意**：颜色由游戏引擎根据 HP 值自动分配，关卡文件中**不要**包含 `color` 字段。

## 设计建议

### 难度递进
- 前期关卡：小网格（16-20列）、低球速（280-320）、宽挡板（88-120）、4-5 条命
- 中期关卡：中等网格（20-26列）、中等球速（320-360）、中等挡板（80-88）、3 条命
- 后期关卡：大网格（26-30列）、高球速（360-400）、窄挡板（70-80）、3 条命

### 砖块密度
- 简单关卡：60-120 个砖块
- 中等关卡：120-250 个砖块
- 困难关卡：250-550 个砖块

### 注意事项
- 砖块是正方形渲染的，宽高相等
- 网格坐标从 `(0, 0)` 开始
- 不可摧毁砖块不计入通关条件（只需消灭所有可摧毁砖块即可通关）
- 火球道具可以一击摧毁任何可摧毁砖块（包括 HP=4 的）
- 不可摧毁砖块即使被火球击中也不会被摧毁

## 示例：创建一个简单的十字形关卡

```json
{
  "name": "关卡13 - 十字",
  "gridWidth": 16,
  "gridHeight": 10,
  "ballSpeed": 300,
  "paddleWidth": 90,
  "lives": 4,
  "bricks": [
    { "row": 0, "col": 7, "hp": 2 },
    { "row": 0, "col": 8, "hp": 2 },
    { "row": 1, "col": 7, "hp": 1 },
    { "row": 1, "col": 8, "hp": 1 },
    { "row": 2, "col": 7, "hp": 1 },
    { "row": 2, "col": 8, "hp": 1 },
    { "row": 3, "col": 3, "hp": 1 },
    { "row": 3, "col": 4, "hp": 1 },
    { "row": 3, "col": 5, "hp": 1 },
    { "row": 3, "col": 6, "hp": 1 },
    { "row": 3, "col": 7, "hp": 2 },
    { "row": 3, "col": 8, "hp": 2 },
    { "row": 3, "col": 9, "hp": 1 },
    { "row": 3, "col": 10, "hp": 1 },
    { "row": 3, "col": 11, "hp": 1 },
    { "row": 3, "col": 12, "hp": 1 },
    { "row": 4, "col": 7, "hp": 1 },
    { "row": 4, "col": 8, "hp": 1 },
    { "row": 5, "col": 7, "hp": 1 },
    { "row": 5, "col": 8, "hp": 1 },
    { "row": 6, "col": 7, "hp": 2 },
    { "row": 6, "col": 8, "hp": 2 }
  ]
}
```

添加此文件后运行 `node build.js` 即可在游戏中看到新关卡。
